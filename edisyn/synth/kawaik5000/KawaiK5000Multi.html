Kawai K5000

Multimode Patch Editor

This editor will work with the Kawai K5000S, KawaiK5000R, and , but it does not support banks B or G on the K5000W.

Communicating with Edisyn

Turn OFF Protect. Make sure that Receive Program and Receive Exclusive are both turned on. Set Program Receive to NORMAL. Set the Unit Channel (System Channel) of your K5000 to the same channel as you've set up Edisyn (perhaps channel 1?).

About Sending to Current Patch

The Kawai K5000 stupidly cannot receive a patch and store it in its current temporary memory (nor transmit the same). This is a huge failing as Edisyn needs to do this for many things, everything from undo/redo to randomize to hill-climbing. There are two ways around this, neither of which is good. First, Edisyn could write to a "scratch patch slot" (such as D128), then load this patch slot into current memory. This is what the Kawai K5 editor does. This would be fine except that the K5000 uses Flash RAM for its storage, so repeatedly writing to a patch slot is not a good idea: it'll burn your Flash out. So that's out: I won't damage people's synthesizers. Second, Edisyn could transmit every single parameter to the K5000 in sequence. However to transmit a single parameter takes 14 bytes, and there are upwards of 5000 parameters. This can take as much as 25 seconds to transmit!

So this is a problem. At present our only real option is to NOT send to current memory unless you manually select "send to current memory". This will make the hill-climber and morpher worthless, as well as nudging and morphing, and mixing multiple patches in the librarian. But it's the only reasonable option.

Gotchas

  • See the discussion About Sending to Current Patch above: it is a massive and problematic gotcha.

  • If you press the Show button on a Section, it will switch to that patch and display it. However there is no way to go back to the multimode patch. You will need to manually change to Multi mode and navigate to the patch.
