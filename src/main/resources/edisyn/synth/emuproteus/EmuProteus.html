E-Mu Proteus 1, 1 XR, 2, 2 XR, 3, 3 XR, and 1+Orchestral

Single Patch Editor

This editor will work with the E-Mu Proteus 1, 2, 3, and 1+Orchestral, and with various XR versions, but you will need to set a switch to indicate which one you are using, as this will affect which patch numbers you can load and what instruments are available.

Communicating with Edisyn

Set the MIDI ID to the same you'll be setting in Edisyn (typically 0).

Using Different ROMs/soundsets

This patch editor is designed to work with standard Proteus models, not ones with changed or additional soundsets. If you have a common configuration that uses different instrument soundsets, send me email (sean@cs.gmu.edu) and we might discuss how the patch editor could be extended to accommodate them. It's not for the faint of heart, however.

Gotchas

  • The different Proteus series machines have different instruments and different numbers of instruments. If you load or receive a patch meant for a different Proteus than you have set, then the instruments will not load properly and will be set to "No Instrument". (For example, if you have set Edisyn to Proteus 1 and you load a patch meant for a Proteus 3).

  • The Proteus series cannot send to the current patch -- they can only write patches, or send individual parameters. But sending to the current patch is a critical function. Instead of (slowly) sending all parameters, Edisyn instead will write to patch 127. This means that you will overwrite this patch number every time you "send" the current patch.

  • While the Proteus 2 and 3 can set the Chorus to OFF or to 1-15, the Proteus 1 can only actually set the Chorus to OFF and ON.

  • You cannot request the current patch on the Proteus.

  • In order to change patches (to request a new one or to send or write a patch), Edisyn must modify the Proteus's Program Map table: but there is no way to do this with precision. As such, Edisyn just obliterates the table in the process. I hope you didn't have anything in there that you wanted to keep.
