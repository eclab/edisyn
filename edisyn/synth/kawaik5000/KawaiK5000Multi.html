Communicating with Edisyn

Turn OFF Protect. Make sure that Receive Program and Receive Exclusive are both turned on. Set Program Receive to NORMAL. Set the Unit Channel (System Channel) of your K5000 to the same channel as you've set up Edisyn (perhaps channel 1?).


How K5000 Memory Works and its Implications

The K5000 memory structure is weird, and how it presents itself to you on the K5000 screen is quite different from how it presents itself to you using an editor. So it's very important to understand how it works with an editor.

In a traditional synthesizer, you have a permanent storage of patches, from which you call forth a single patch to play and/or edit. To do this, the patch is copied to current working memory; and it may be written back to the permanent storage later.

In the K5000 you also have a permanent storage of patches, but when you boot up the machine, it copies every single patch in every bank from the permanent storage to the current working memory. If you edit a patch manually on the synthesizer, then before you exit and switch to a new patch you are required to either write the patch back to permanent storage, or to reload from permanent storage, erasing your changes. Thus either of these operations (in different ways) bring the current working memory in sync with the permanent storage.

But if you edit a patch over Sysex from an editor like Edisyn, the Kawai K5000 behaves very differently. You can modify any patch you like in current working memory, or multiple ones, and you're not required to write any of them to permanent storage until you feel like it, if at all. If you reboot the K5000 without writing them, your changes are gone.

This poses some major opportunities for confusion. Edisyn deals with this by doing some things to maintain consistency:

  • Edisyn always resets (reloads from FLASH) whenever you request a new patch.

  • When you select "Write to Patch...", Edisyn resets (reloads from FLASH), then writes the patch to current working memory, then backs up (writes all patches to FLASH). This is a slow process, and the machine will hang for perhaps 15 seconds!

  • When you write one or more patches in the Librarian, or write a whole bank, Edisyn will likewise reset (load from FLASH), write all the patches (slow), and then back up (write all patches to FLASH, again taking 15 more seconds).

  • When Edisyn updates the current patch (such as after undo, redo, randomization, etc., or when you select "Send to Current Patch", in some situations it doesn't know what the patch number and bank are, so it will switch to and temporarily use A001 as a "scratch patch slot" to send the data to. Don't pay that any mind.

About the Menu

The Kawai K5000 menu has the following options:
  • Reset (Reload from Flash) will replace all patches in current working memory with fresh copies from the permanent store. Any edits you have made will be lost.

  • Backup (Write to Flash) will copy all patches in current working memory to the permanent store. You might do this after clearing the bank (see above), or in certain situations if you have unchecked Always Backup after Write to Patch/Bank (see below).

  • Perform Backup and/or Reset after Writes and Requests is checked by default. When checked, then when Edisyn writes a patch (via Write to Patch... or the various Librarian's Write Selected Patches to Synth or Write Bank to Synth) it first does a RESET, then writes the patch(es) or Bank, then does a BACKUP. Additionally, when you request a patch, a RESET is done first. This all guarantees that Edisyn's "write" behavior actually writes to FLASH. However there are situations when you don't want this.

    For example, imagine you want to load a bank from disk, then use Edisyn to write a patch or two to select slots, replacing them, then save back out to disk. What would happen is: you load the bank from disk into current working memory. Then in the process of writing one or two patches, Edisyn replaces the current working memory with the permanent (Flash) store, then writes those patches out the permanent store. If you saved back out to disk, you'd have erased your bank: and the new would have been written to permanent memory where you didn't want them! Instead, temporarily uncheck this menu, then load from disk, perform a write with Edisyn (which now will just put the patches in permanent store), write to disk, and check the menu again. You could perform a RESET for good measure to restore the patches from permanent store if you cared.

    Another situation is if you want to load a few patches from your hard drive to write to permanent store without using the Librarian, but you don't want to burn Flash RAM too much. Uncheck the menu, then load each patch into the slot you want, update each patch separately into current working memory with "Write to Patch...", then at the end, perform a BACKUP to write everything to permanent store. Then recheck the menu again.

    If you uncheck this menu, Edisyn will do three things. First, Edisyn's "Write to Patch..." and various Librarian "Write" menu options will just write the patch to current working memory. It won't RESET first, and it won't BACKUP afterwards. Second, Edisyn's "Request Patch" commands will not perform a RESET first. Third, all operations which send to current memory will be disabled: this includes the Send to Current Patch menu option, as well as updating current memory whenever you perform an undo/redo/randomize/hill-climb/morph/etc. Unchecking this menu isn't for performing edits, it's for writing to the disk from Edisyn. So you probably shouldn't change parameters -- they'll still modify the current patch.



Display Updates

Some parameter changes will update the display on the K5000 so you'll see them change. Others will not. It's fairly arbitrary. If you update a patch (send to current patch, undo/redo, randomize, etc.), it the result will be displayed until you manually edit. Notably, the patch name won't change on the screen: you can see the change by switching to a different patch, then going back.

Gotchas

  • If you press the Show button on a Section, it will switch to that patch and display it. However there is no way to go back to the multimode patch. You will need to manually change to Multi mode and navigate to the patch.
