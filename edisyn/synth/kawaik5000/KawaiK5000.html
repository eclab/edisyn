Kawai K5000

Single Patch Editor

This editor will work with the Kawai K5000S, KawaiK5000R, and Kawai K5000W, including the E and F expansion banks. However it does not support Bank B on the K5000W.

Communicating with Edisyn

Turn OFF Protect. Make sure that Receive Program and Receive Exclusive are both turned on. Set Program Receive to NORMAL. Set the Unit Channel (System Channel) of your K5000 to the same channel as you've set up Edisyn (perhaps channel 1?).


How K5000 Memory Works and its Implications

The K5000 memory structure is weird, and it's very important to understand how it works with an editor.

In a traditional synthesizer, you have a permanent storage of patches, from which you call forth a single patch to play and/or edit. To do this, the patch is copied to current temporary memory; and it may be written back to the permanent storage later.

In the K5000 you also have a permanent storage of patches, but when you boot up the machine, it copies every single patch in every bank from the permanent storage to the current temporary memory. However if you edit a patch manually on the synthesizer, then before you exit and switch to a new patch you are required to either write the patch back to permanent storage, or to reload from permanent storage, erasing your changes.

But if you edit a patch over Sysex from an editor like Edisyn, the Kawai K5000 behaves differently. You can modify any patch you like in current temporary memory, and you're not required to write any of them to permanent storage until you feel like it, if at all. If instead you reboot the K5000, they're erased. Edisyn cannot issue a write command to write a single patch back to permanent storage, nor can it reload a single patch: there's no sysex command for these. Instead to store a single patch modified by Edisyn, you have to manually edit the patch, make some small parameter change (and maybe change the parameter back), then exit the patch, where you'll be prompted to write it to permanent memory. And there is no way reload a single patch edited by Edisyn.

What Edisyn can do is to write every single patch, of every bank to permanent storage. This is done by issuing a Backup from the K5000 menu in Edisyn, or from the unit itself. And Edisyn can also reload every single patch, of every bank from permament storage, overwriting all changes you've made to any of them in current temporary memory. This is done by issuing a Reset from Edisyn or on the machine.

Critically note that permanent storage is FLASH RAM, and FLASH will eventually burn out, perhaps after 10,000 writes, so we don't want to tax it that much. You can read FLASH RAM forever. Since a BACKUP is spread out over the full Flash RAM, it's not much more taxing than a single WRITE, so don't worry too much about using a BACKUP in lieu of a WRITE. But if you wanted to be ultra conservative, you could always manually WRITE a single patch you edited in Edisyn, using the method described earlier. At any rate, if you used Edisyn to upload an entire new Bank, you should BACKUP that Bank to save it to permanent storage.

All this means that Edisyn's "Write to Patch..." menu option doesn't write the patch to permanent memory. It just writes it to current temporary memory. You will have to issue a WRITE from there, or a BACKUP. This also means that the "Send to Current Patch" menu option (or any other procedure which updates the current patch, such as when you undo, or redo, or randomize, or hill-climb, or morph, or nudge, etc.) is just doing the same thing as "Write to Patch..." for the current patch! Furthermore, if Edisyn needs to update the current patch and it doesn't know the current bank and number, it has to do something, so it'll just assume it's A001, overwriting what's there. You can fix that by first issuing a "Write to Patch..." menu, which sets the bank and patch number later used by patch updates; or reading the patch in with "Request Patch...". Also the K5000 has no option to request the current patch, so Edisyn has that disabled.

What a mess. Why they did this I have no idea.

Display Updates

Some parameter changes will update the display on the K5000 so you'll see them change. Others will not. It's fairly arbitrary. If you update a patch (send to current patch, undo/redo, randomize, etc.), it the result will be displayed until you manually edit. Notably, the patch name won't change on the screen: you can see the change by switching to a different patch, then going back.

About the Tabs

The K5000 has many parameters. These parameters are organized as follows. First there are the General parameters for the entire patch. Then there are up to six Sources, each of which produces its own sound. You specify the number of Sources in the Sources category in the General Tab. This will change the tabs in the editor. You can also specify which sources are playing and which are muted.

Each Source can be an Additive or PCM (digital sample) source. A digital sample source consists of a single tab called "S" (as in S1, S2, and so on). An additive sample source also has the "S" tab, but it also has two more tabs called "H" (for "Harmonics") and "E" (for "Envelopes"). These are the so-called Additive Wave Kit, and only appear when the Source Type is set to "Additive". You set the Source Type in the DCO category under the "S" tab.

If you set the Source Type to "PCM", you will also be able to set the PCM Wave in question.

About the Dials

The General tab has a Category called "Dials". These aren't part of the patch: they're just a convenience for you. Changing these dials changes any of the 16 Macro Controller Dial settings on your K5000.

About Morphing

The K5000 has a special morphing facility called "Morf". In this facility, you set various interpolation options, then tell the synthesizer to morph the harmonics of up to four different patches, creating an interpolated harmonics envelope set. You can set these parameters in Edisyn as well, in the "Morf" category, and you can instruct your synthesizer to perform the morphing by pressing the "Execute" button in Edisyn. However, after you have performed morphing, the results are only stored on the synthesier, and are unfortunately not given to Edisyn. To load the results into Edisyn, you'll need save the patch manually on your synthesizer, then have Edisyn request the patch. This is the only option: the K5000 has no way to request the current patch memory. Your most recent settings of morphing parameters are stored with your patch, presumably to make it easy to execute a morph a second time. There is also a "Show Morf" checkbox in Edisyn, because this is a parameter in your patch. All it does is instruct your synthesizer to display the Morf screen instead of the regular harmonics screen. It's not very useful.

About the Harmonics Display

The "H" (Harmonics) tab and "E" (Envelopes) tab for each source contain a number of displays for harmonics, formants, and envelope settings. You can set these by Number, then Value, or you can directly modify the sliders in the display itself. If you change a slider, only that slider can be set. But you can "scrub" across sliders by changing the slider with the right mouse button, or by holding down SHIFT and dragging.

The "Loop" display sliders each have three settings: OFF (the lowest setting), LP1 (the middle setting), and LP2 (the highest setting).

About the Menu

The Kawai K5000 menu has the following options:
  • Constrain Harmonics  This locks the harmonics you can edit with the sliders to a certain subset of harmonics, to make it easier to find and edit them. The other harmonics will be displayed in gray. You can still set any harmonics with the Number dial.

  • Load Wave Into...  You can load a WAV file as harmonics into the soft or loud harmonics of any source. The WAV file should store a single-cycle wave. Single-cycle waves must be stored as 16-bit WAV files no longer than 2048 samples. A good source of single-cycle waves is the AdventureKid Waveforms collection (https://www.adventurekid.se/akrt/waveforms/adventure-kid-waveforms/), which has thousands of waveforms, all free.

  • Allow Envelope Display Mouse-Over  This prevents mouse-over on the harmonics envelope displays to update which harmonics envelope you're editing. If you're fed up with accidentally mousing over an envelope display when you're changing a dial and changing the envelope number, this menu item is for you.

About the "Shaping" Categories

These categories contain various utilities to shape the Harmonics and Harmonics Envelope Levels. The Dials hold common harmonics (sawtooth, triangle, square, various pulse) plus a large number of harmonics drawn from patches on the original Kawai K5 synthesizer. If you select a value on the Dial, it will set the appropriate Harmonics. The Dials are not part of the patch: they're just utility devices and their value is only meaningful as you're dialing them. Similarly, the associated Buttons allow you to perform various editing operations. Note that both the Dials and the Buttons are constrained by your selection of harmonic Constraints (see the Menu).

How to Bulk-Set Envelope Displays

  • You can copy and paste envelope displays (for harmonics, formants, or harmonic envelopes) to appropriate tabs in the same tab or on other tabs. To do this, just click on the Category name above the envelope display and choose "Copy Category" or "Paste Category" as appropriate.

  • You can set all the parameters in an envelope display (for harmonics, formants, or harmonic envelopes) to the same value. To do this, first set one parameter to the value you want, then click on the Category name above the envelope display and choose "Distribute".

  • You can make all the parameters in an envelope display mutable, or immutable, by clicking on the Category name and choosing "Make Category Parameters Mutable" or "Make Category Parameters immutable". This is important because normally you can't an envelope mutable or immutable via "Edit->Edit Mutable Parameters".

  • You can copy all the parameters of from one tab to another. For example, you can copy the "E1" tab to the "E4" tab. To do this, go to the first tab, then select "Copy Tab" under the Edit menu, then go to the second tab, and select "Paste Tab" under the Edit menu.

About The Librarian

Kawai K5000 bank downloads are gigantic, because Kawai K5000 patches are gigantic. They are so large that Edisyn cannot use Kawai's bank upload sysex message to upload a bank, because it exceeds Java's MIDI message maximum size. Instead, to upload a bank, Edisyn will attempt to erase the bank, then upload each patch in turn. This isn't too slow, but it's suboptimal. It will take a while for the unit to respond to all of them however: be patient. It'll feel sluggish or hang until it's done.

Edisyn can receive the gigantic bank messages from the K5000. It could take as much as a minute or more to download a bank using the fastest method, which is via "Request Bank from Synth". However you will get NO feedback until the bank finally arrives. Using "Download Bank from Synth" or "Download Selected Patches from Synth" is slower, but you'll see the patches as they arrive.

If you download using "Download Bank from Synth" or "Download Selected Patches from Synth", when Edisyn reaches the null patches at the end of the bank, the K5000 never tells Edisyn this: instead it just sits there, and Edisyn will sit there as well until it times out. At that point, you might want to manually press the "Stop Download" button in the Librarian.

If you upload a range of patches, it'll probably work fine, but if it exceeds the avaiable bank memory, it might fail. The K5000 provides Edisyn with no way to erase one patch or a range of patches. I'll try to improve things here.
